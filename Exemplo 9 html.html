Exemplo 9 HTML Semântico e uso da tag <a>

As tags semânticas em HTML, como header, footer, article =, nav e aside, descrevem o significado e a função do contéudo que o contêm, não apenas sua aparência. Elas são cruciais para acessibilidade ajudando leitores e mecanismos de busca no entendimento do conteúdo do site.

Este documento é um exemplo do uso das tags semânticas.

React: Uma Biblioteca JavaScript para Interfaces de Usuário Modernas React é uma biblioteca JavaScript de código aberto, mantida pelo Facebook, utilizada para construir interfaces de usuário (UI) interativas e dinâmicas para aplicações web. Lançada em 2013, a tecnologia se tornou uma das mais populares no desenvolvimento front-end devido à sua eficiência e flexibilidade. A principal característica do React é sua arquitetura baseada em componentes. Essa abordagem permite que os desenvolvedores dividam a interface em pequenas partes isoladas e reutilizáveis, chamadas de componentes. Cada componente gerencia seu próprio estado e lógica, o que simplifica o desenvolvimento e a manutenção de aplicações complexas.

Blazor - C#

Blazor é um framework de código aberto desenvolvido pela Microsoft que revoluciona o desenvolvimento web ao permitir que desenvolvedores criem aplicações web interativas usando C# e .NET, em vez do tradicional JavaScript. Lançado como parte da plataforma .NET, o Blazor permite a criação de interfaces de usuário (UI) ricas e componentizadas,

Principal característica

principal característica do Blazor é: Usar a linguagem C# para construir o front-end de uma aplicação web, substituindo a necessidade de usar JavaScript.

  • Blazor WebAssembly (Wasm): Neste modelo, a aplicação C# e o runtime do .NET são compilados para WebAssembly, um formato de código binário que roda diretamente no navegador do cliente com desempenho próximo ao nativo.[5][6] Uma vez que a aplicação é carregada, ela pode funcionar de forma independente, inclusive offline, sem a necessidade de uma conexão constante com o servidor.[7][8]
  • Blazor Server: Aqui, a aplicação é executada no servidor, enquanto o navegador atua como um terminal leve.[9] As interações do usuário são enviadas para o servidor através de uma conexão em tempo real (usando SignalR), que processa os eventos e envia de volta as atualizações da interface.[9][10]
